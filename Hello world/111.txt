这个是澳大利亚的部队部署文件，前面的division_template块是部队的模板，其中包含着你要用到的能决定部队模板的name，以及代表兵种组成的多个坐标；

具体的任务需求是这样的：首先向用户询问三位的国家tag，然后根据用户输入的tag（转为全大写），在F:\Steam\steamapps\common\Hearts of Iron IV\history\units 这个路径中寻找形如TAG_1936.txt的文件，读取所有其中division_template块，并写入到F:\Documents\Paradox Interactive\Hearts of Iron IV\mod\World_War_Online\Hello world\history\units\TAG_1905.txt中，其中TAG为用户输入的tag;
之后，向用户询问需要多少个师，然后循环地向用户询问可以放置的地块id，输入q结束。
我希望你生成的division块模板如下
division= {
		division_name = {
				is_name_ordered = yes
				name_order = 7
		}
		location = 688  # Salzburg
		division_template = "Alpenjäger Division"
		start_experience_factor = 0.2
		start_equipment_factor = 0.3

	}
	这样的形式，其中name_order随着这个模板出现次数递增。
	多个division块中，location的数字是部队具体部署的地块id。
	start_experience_factor固定设为0.2
	start_equipment_factor为0.3~0.9之间的随机数，保留一位小数。
	然后不断地写division块，且随机地选择不同的、regiments块里只由infantry组成的部队模板名来填充division块里的division_template，location则随机地选择用户输入的多个地块id中的一个，某个模板每被选择一次，name_order就增加1。如此循环师的数量直到达到用户输入的师数。
	最后补上生产的内容，类似下面的：
	instant_effect = {
